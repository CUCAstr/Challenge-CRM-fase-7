# Challenge WTC - 2ª Etapa (Sprint 2)

## Introdução
A Sprint 2 foca na construção do backend real para substituir os mocks da etapa anterior. Objetivos:
* APIs completas.
* Integração com aplicativo nativo.
* Troca de mensagens.
* Mídia & Deeplinks (links interativos).
* Governança e observabilidade (auditoria e logs).
* Usabilidade com atualização em tempo real.

O projeto visa transformar o protótipo em uma plataforma robusta de comunicação integrada ao CRM do WTC.

## Entregáveis

### Entregas Obrigatórias
* **Projeto Backend (.zip):**
  * Código-fonte organizado por camadas/módulos.
  * Arquivo `read.me` com instruções de execução.
* **Projeto Aplicativo (.zip):**
  * Pasta compactada com artefatos de funcionamento.
  * Integração com backend (APIs REST).
  * Executável: Arquivo APK (Android) ou IPA (iOS).
* **Apresentação (.PDF ou .PPT):**
  * Nome e RM dos integrantes.
  * Diagrama de arquitetura do backend.
  * Especificação de endpoints (rota, método, payload, resposta).
  * Detalhamento do modelo de dados NoSQL.

### Critérios de Avaliação
| Critério | Peso |
| :--- | :--- |
| Implementação técnica funcional | 50% |
| Integração aplicativo com o backend | 15% |
| Apresentação e documentação | 15% |
| Qualidade do código e boas práticas | 10% |
| Criatividade e inovação | 10% |
